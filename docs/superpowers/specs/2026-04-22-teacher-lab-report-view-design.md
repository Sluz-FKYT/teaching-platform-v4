# 教师端动态实验报告视图设计

## 目标

为教师端实验报告批改队列页增加一个“实验报告”按钮。教师点击后以**新标签页**打开一张**纯报告视图**，页面不展示教师后台壳层、筛选器、批改控件或其他管理 UI，只按标准实验报告模板动态渲染当前实验、学生与其**最新一次提交**对应的报告内容。

本次设计目标是复刻老系统“打开一张标准实验报告页面”的使用感受，而不是扩展现有教师批改台。

## 背景与现状

当前仓库中已经存在两类与“实验报告”相关的教师端页面：

1. `frontend/src/views/teacher/LabReports.vue`
   - 这是教师端“实验报告批改队列”页面。
   - 页面里已经按实验、学生和状态展示了提交记录，并提供“查看评分 / 进入批改 / 评分（填空）”等操作。

2. `frontend/src/views/teacher/LabReportDetail.vue`
   - 这是逐题评分与提交总评的批改详情页。
   - 页面核心目标是批改，不是实验报告模板展示。

因此，当前系统虽然有“实验报告”相关命名和真实数据链路，但其语义偏向**批改工作台**，与用户要求的“纯报告只读页”不一致。

用户本次明确排除了以下方向：

- 不要做新的批改流程改造
- 不要保留教师后台样式壳层
- 不要加花哨 UI
- 不要做额外“实验报告列表”或“报告管理界面”
- 不要把标准实验报告做成评分台的另一个模式

## 设计结论

本次采用以下方案：

1. **保留现有 `LabReports.vue` 作为入口页**，不推倒重来。
2. 在 `LabReports.vue` 每条实验报告记录的操作区新增一个 **“实验报告”** 按钮。
3. 点击按钮后使用新标签页打开一个**独立的纯报告页面**。
4. 该页面不复用教师后台布局，而是只渲染实验报告正文。
5. 后端新增一个面向教师的聚合接口，按 `labId + studentId` 解析该学生在该实验下的**最新提交记录**，并动态组装报告数据。
6. 报告不落库，不保存为独立文档，不做历史版本管理，不做导出能力。

## 一、入口设计

### 1. 页面位置

按钮入口放在：

- `frontend/src/views/teacher/LabReports.vue`

具体位置为每条报告记录右侧操作区，和当前的：

- `查看评分 / 进入批改`
- `评分（填空）`

并列展示一个新按钮：

- `实验报告`

### 2. 按钮行为

点击“实验报告”按钮后：

1. 不在当前页内联展开
2. 不弹窗
3. 不在当前标签页跳转替换批改队列
4. 使用新标签页打开报告页

推荐行为：

```ts
window.open(reportUrl, '_blank', 'noopener,noreferrer')
```

### 3. 上下文参数

按钮点击时传递当前列表项的：

- `labId`
- `studentId`

报告页和后端都不以“提交记录 ID”作为业务主语义，而是以：

> 某个实验 + 某个学生

再在服务端解析这个组合的**最新提交**。

---

## 二、报告页设计

### 1. 页面定位

报告页是一张**纯内容页**。

它不是教师后台工作台的一部分，而是更接近老系统中的“报告查看窗口”。页面打开后不应出现：

- 侧边栏
- 顶部导航
- 面包屑
- 操作工具栏
- 卡片式教师后台摘要区
- 评分输入框
- 任何批改按钮

页面只保留报告正文结构。

### 2. 视觉原则

风格尽量贴近用户给出的老系统截图：

- 白底
- 正文文档感排版
- 顶部居中标题
- 左右分布的基础信息
- 分章节展示实验目的、实验内容、实验步骤、实验小结
- 不做教师端管理工作台式装饰

允许使用最少量的样式来保证可读性，但不能让页面重新变成后台管理 UI。

### 3. 报告页路由原则

报告页需要独立路由，但页面本身不套教师布局壳层。

建议语义：

- `/teacher/labs/:labId/report/:studentId`

说明：

- 路由仍处于教师权限保护下
- 但渲染组件时应跳过教师后台外层壳层
- 路由仅承担权限与参数语义，不代表页面要长成后台页

---

## 三、数据装配设计

报告页是运行时动态拼装结果，不新增“实验报告正文”持久化存储。

### 1. 数据来源分层

报告内容由三部分组合而成：

1. **实验定义**
   - 实验名称
   - 实验要求
   - 实验内容
   - 实验步骤定义

2. **学生与班级信息**
   - 班级
   - 学号
   - 姓名

3. **学生最新提交 + 当前评分结果**
   - 每个步骤的最新作答
   - 当前总评
   - 教师评语（如果已有）

### 2. 最新提交规则

本次明确采用：

> 报告永远基于该学生在该实验下的**最新一次提交记录**生成。

这意味着：

- 如果该学生存在多次提交，只取最新一条
- 不支持在报告页切换历史提交版本
- 不支持报告版本管理
- 列表中的单条记录也不应强依赖旧的 `LabSubmission.id` 作为报告主入口语义

### 3. 课程名称规则

课程名称固定写死为：

- `软件设计与体系结构`

不从课程表或班级表动态推导。

### 4. 日期规则

报告页顶部日期使用当前日期，而不是提交日期或批改日期。

### 5. 教师评定规则

“指导教师评定”区域显示规则如下：

- 若该最新提交已有总分，则显示总分
- 若尚未出分，则该位置留空
- 不显示 `0`
- 不显示“未批改”提示文案

这是为了保持与老系统页面观感一致。

---

## 四、报告内容结构设计

报告页正文按以下结构组织：

### 1. 标题区

- 标题：`南昌航空大学实验报告`
- 日期：当前日期

### 2. 基础信息区

展示以下字段：

- 课程名称：`软件设计与体系结构`
- 班级
- 实验名称
- 学号
- 姓名
- 指导教师评定
- 签名（留空展示位）
- 同组人（若无数据则留空展示位）

### 3. 实验目的

映射：

- `Lab.description`

即当前实验的“实验要求”。

### 4. 实验内容

映射：

- `Lab.experimentContent`

### 5. 实验步骤

按步骤序号升序展示。

每一步至少显示：

- 步骤序号
- 步骤标题
- 步骤题干 / 说明
- 学生答案

关于答案显示原则：

- 选择题：展示最终选项内容或答案文本结果
- 填空题：展示学生填空结果整理后的文本
- 简答题 / 文本题：展示原始文本答案
- 代码题：展示代码文本，保留换行
- 若该步无答案，则对应答案区域留空

### 6. 实验小结

若当前系统已有学生实验小结内容，则显示在“实验小结”章节；若没有内容，则章节保留标题并留空，避免为此新增新的数据模型。

## 五、后端接口设计

### 1. 接口目标

后端需要新增一个专门面向教师报告页的聚合接口，避免前端自己调用多个旧接口拼装。

建议接口：

- `GET /api/v1/teacher/labs/{labId}/report-view/{studentId}`

### 2. 接口职责

该接口应一次性返回报告页渲染所需的完整数据，包括：

- 标题区数据
- 学生与班级信息
- 实验基础信息
- 最新提交记录摘要
- 按步骤整理后的作答内容
- 当前总评 / 教师评语

### 3. 后端聚合规则

服务端需要：

1. 校验当前教师对该实验有访问权限
2. 以 `labId + studentId` 查询该学生在该实验下的最新提交
3. 查询该实验步骤定义
4. 查询该最新提交下的步骤答案
5. 组装成报告投影视图 DTO

### 4. 空态策略

若教师打开报告页时该学生尚无任何提交，则返回清晰错误或空态，不伪造报告内容。

## 六、前端实现边界

### 1. 需要新增

- `LabReports.vue` 中的“实验报告”按钮
- 一个新的前端 API 调用函数
- 一个新的纯报告组件/页面
- 一个不套教师后台壳层的路由出口

### 2. 明确不做

本次第一版不做以下内容：

- PDF 导出
- Word 导出
- 报告打印优化按钮
- 报告落库
- 报告历史版本切换
- 学生端查看报告
- 报告页中编辑评分
- 报告页中继续批改

这能保证本次改造聚焦于“只读动态实验报告视图”本身。

---

## 七、与现有页面的关系

### 1. `LabReports.vue`

保留为教师端报告相关入口页，但它的职责仅限于：

- 显示队列
- 提供进入批改
- 提供进入填空评分
- **新增进入实验报告只读页**

### 2. `LabReportDetail.vue`

继续保留为批改详情页，不承担实验报告模板展示职责。

原因：

- 该页已有逐题评分与提交总评职责
- 与标准实验报告展示的语义冲突
- 强行复用会把“只读报告”与“评分工作台”耦合到一起

因此本次采用“入口复用，报告页新增”的策略。

---

## 八、验证标准

实现完成后，需要满足以下验收标准：

1. 教师在 `LabReports.vue` 的每条记录上都能看到“实验报告”按钮。
2. 点击按钮后会以**新标签页**打开报告页。
3. 报告页不显示教师后台壳层或任何管理型 UI。
4. 页面内容结构与老系统标准实验报告相近。
5. 报告数据来自该学生在该实验下的**最新提交**。
6. 课程名称固定显示为 `软件设计与体系结构`。
7. 未出分时，“指导教师评定”位置留空。
8. 报告页正文能正确展示实验目的、实验内容、步骤与学生答案。
9. 现有批改页与评分流程不被破坏。

## 九、风险与注意事项

1. 当前现有教师报告列表和详情链路偏向 `LabSubmission` 提交记录级别，而不是 `labId + studentId` 最新提交聚合级别，实现时不能直接照搬旧详情接口。
2. 若直接把新报告页塞进教师后台壳层，会违背用户对“纯报告页面”的要求。
3. 报告按钮必须以“新标签页”打开，否则会打断教师在批改队列中的上下文。
4. 报告页不能为了图省事复用评分台结构，否则会引入不必要的按钮和后台装饰。
5. 代码题与长文本答案的渲染必须保留换行，避免报告内容可读性丢失。

## 十、最终决策摘要

本次实验报告功能不沿用现有批改详情页形态，而是：

- 在 `LabReports.vue` 记录操作区新增“实验报告”按钮
- 新开标签页打开一个**纯报告页面**
- 页面不带教师后台 UI，只渲染标准报告内容
- 数据由新的教师端聚合接口按 `labId + studentId` 解析**最新提交**后动态返回

这是满足当前需求且对现有批改链路侵入最小的方案。
