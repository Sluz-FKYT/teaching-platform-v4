# 实验步骤配置页三栏工作台重构设计

## 目标

将教师端实验步骤配置页从“整页长表单 + 低密度大卡片”重构为**固定视口的三栏工作台**，满足以下目标：

1. 页面不再以整页滚动作为主要交互方式。
2. 左侧显示实验步骤轨道，中间显示题型驱动的主编辑区，右侧显示分值与执行约束。
3. 左、中、右三栏各自拥有独立滚动容器。
4. 中间主编辑区尽量复用题库新增题目页已经验证过的 typed-editor 交互。
5. 保存实验步骤时可选“同步创建到题库”，但题库题目与实验步骤仅在本次保存时共享编辑内容，首次同步后独立维护。

## 背景与现状

当前教师端实验步骤配置页入口为 `frontend/src/views/teacher/LabSteps.vue`，路由为 `/teacher/labs/:id/steps`。

页面已经具备“左侧步骤列表 + 右侧编辑区”的雏形，但仍存在以下问题：

1. 顶部 hero 和上下文卡片占据大量垂直空间，降低首屏有效编辑面积。
2. 页面整体仍以外层主内容区滚动为主，没有真正的分栏 scroll ownership。
3. 中央编辑区和右侧规则区虽然拆开，但仍是传统纵向堆叠，用户在步骤切换、规则配置、分值确认之间频繁失去上下文。
4. 题型编辑逻辑与题库新增题目页存在重复趋势，但目前没有形成共享 typed-editor 边界。
5. 页面尚不支持“保存实验步骤时可选同步创建到题库”的业务动作。

## 设计结论

本次采用以下总体方案：

- 将 `LabSteps.vue` 重构为**三栏工作台**。
- 抽出可被题库页与实验步骤页共同消费的 **typed-editor 共享层**。
- 实验步骤页和题库页各自保留自己的保存适配层，不共享最终 payload。
- 右栏新增“保存时同步创建到题库”勾选项。
- 同步到题库采用**首次同步后独立维护**语义，不建立持续绑定，也不默认覆盖已有题库题目。

该方案对应用户确认的“方案 A”。

## 一、页面结构设计

### 1. 页面整体布局

页面改为两层结构：

1. **压缩头部（header strip）**
2. **固定高度的 workbench 主体**

头部仅保留：

- 返回实验列表
- 实验标题 / ID / 题项数 / 总分
- 当前编辑对象简要状态
- 主保存按钮 / 新增题项按钮

删除当前 hero 区中冗长标题说明和三张高卡片，只保留一行上下文 chips，避免头部吞噬可用高度。

### 2. 三栏分区

#### 左栏：步骤轨道

职责：导航与切换。

包含内容：

- 步骤列表
- 每个步骤的序号、标题、题型、分值、配置状态
- 新增题项入口
- 删除当前步骤的快捷动作（如保留）

要求：

- 固定窄宽度。
- 顶部标题与新增按钮固定。
- 仅步骤列表 body 滚动。
- 高密度展示，去掉过多说明性文案。

#### 中栏：typed-editor 主编辑区

职责：编辑当前步骤的核心内容和题型结构。

包含内容：

- 步骤标题
- 步骤说明 / 题干主体
- 题型选择
- 按类型驱动的结构化编辑器

题型范围：

- `SHORT_ANSWER`
- `TEXT`
- `SINGLE_CHOICE`
- `MULTIPLE_CHOICE`
- `TRUE_FALSE`
- `FILL_BLANK`
- `CODE`

要求：

- 中栏是整个页面的主区域，宽度最大。
- 中栏内部采用“顶部元信息 + 可滚动编辑主体”。
- 切换步骤时，中栏骨架不变，仅内容切换。

#### 右栏：属性与约束

职责：编辑与当前步骤相关的执行约束和保存附加动作。

包含内容：

- `stepScore`
- `allowPaste`
- 当前配置摘要
- 完成度提示
- “保存时同步创建到题库”勾选项

要求：

- 右栏宽度收敛但足够容纳表单控件。
- 顶部状态摘要固定。
- 仅右栏内容 body 滚动。

## 二、滚动与视口规则

### 1. 滚动归属

本次重构的核心规则如下：

- 页面外层不再作为主要滚动容器。
- `LabSteps` 页面主体必须锁定在视口可用高度内。
- 滚动仅允许出现在：
  - 左栏步骤列表
  - 中栏编辑器主体
  - 右栏属性区主体

### 2. 具体实现原则

1. `layout/index.vue` 或页面根容器需要提供明确的高度边界。
2. `LabSteps.vue` 内部 workbench 区域需采用 `min-height: 0` / `overflow: hidden` 组合来阻断整页滚动继续传导。
3. 三栏内部各自使用 `display: flex; flex-direction: column;`，由 body 区承担 `overflow: auto`。
4. 不允许再出现“左栏跟着整页滚动”“右栏配置被挤到页面下方”的情况。

### 3. 用户可感知结果

用户进入页面后，无论当前步骤内容多长，都应该始终同时看到：

- 左侧步骤轨道
- 中间当前编辑器
- 右侧分值与约束

页面不会再因为切换到长题干、长代码说明或多空位填空题而变成整页长卷轴。

---

## 三、共享 typed-editor 设计

### 1. 共享范围

共享的不是整个题库页面，而是**typed-editor 核心**。

共享层负责：

- 题型切换
- 不同题型的结构化编辑交互
- hydration（从后端 JSON 转为 UI 编辑态）
- validation（生成统一错误消息）
- draft serialization（生成中间编辑态结果）

页面适配层负责：

- 将共享 draft 转成题库 payload
- 将共享 draft 转成实验步骤 payload
- 控制页面特有字段与附加行为（例如实验步骤的 `stepScore`、`allowPaste`、题库自动编码、同步到题库勾选等）

### 2. 复用目标

题库新增题目页与实验步骤页在以下题型交互上应保持接近一致：

- 单选题：选项列表 + 点击选择正确答案
- 多选题：选项列表 + 点击切换多个正确答案
- 判断题：二元卡片选择
- 填空题：正文插入占位符 + 侧边答案同步
- 简答题：结构化参考答案 / 评分要点输入
- 代码题：参考实现 / 评分要点输入，不涉及学生端代码编辑器

### 3. 为什么不共享最终 payload

题库与实验步骤后端契约不同：

- 题库关注 `code/type/stem/optionsJson/answerJson/analysisText/defaultScore`
- 实验步骤关注 `stepNo/title/content/questionType/answerConfigJson/stepScore/allowPaste`

如果强行共享最终 payload，会让两边字段语义互相污染，并增加未来维护成本。

因此本次采用“共享编辑态，不共享最终接口结构”的设计。

---

## 四、中栏题型编辑器设计

### 1. 单选 / 多选 / 判断

中栏客观题编辑区直接靠近题库页已验证的模式：

- 选项列表为卡片式高密度记录
- 每行支持编辑内容、删除、添加
- 单选题点击一个选项设为答案
- 多选题点击多个选项切换答案
- 判断题使用固定“正确 / 错误”卡片

实验步骤页在保存时将此结构适配为 `answerConfigJson` 中的 choice rule，而不是题库页的 `optionsJson/answerJson` 双字段。

### 2. 填空题

填空题采用题库编辑器中的“正文 + 结构同步”的强模式：

- 中栏正文区支持插入占位符
- 插入按钮会将占位符插入光标位置
- 中栏或右侧副区同步展示每个空位的答案列表
- 删除或调整正文中的空位时，需要尽可能保留已有答案输入

实验步骤页保存时转换成当前后端接受的 fill blank rule config。

### 3. 简答题 / 文本题 / 代码题

主观题共享一个主体编辑器壳：

- 输入参考答案 / 评分要点 / 说明模板
- 文案随题型变化

其中：

- `TEXT` 与 `SHORT_ANSWER` 仍作为实验步骤兼容类型保留
- `CODE` 继续作为实验步骤合法题型存在
- 本次不扩展学生端代码编辑器，不做额外运行或编译配置

---

## 五、右栏设计

右栏承担辅助但关键的“约束与保存附加动作”职责。

建议分成以下区块：

1. **分值设置**
   - 当前步骤分值
   - 在当前实验中的占比提示（如可计算）

2. **执行约束**
   - `allowPaste`
   - 代码题默认语言（若决定放在右栏而不是中栏）

3. **配置完成度摘要**
   - 标题是否填写
   - 内容是否填写
   - 题型规则是否齐全
   - 当前保存将执行“创建 / 更新”哪种动作

4. **题库同步**
   - 勾选项：`保存时同步创建到题库`
   - 说明文案：强调“仅本次同步创建，后续独立维护”

右栏不应继续保留大面积解释卡片，而应压缩成短标签、状态项和可执行配置。

## 六、保存流程设计

### 1. 基础保存流程

点击保存时执行以下顺序：

1. 运行共享 typed-editor 校验。
2. 若存在校验失败，统一在中栏顶部显示消息，并提示首条错误。
3. 生成实验步骤 payload：
   - `stepNo`
   - `title`
   - `content`
   - `questionType`
   - `answerConfigJson`
   - `stepScore`
   - `allowPaste`
4. 调用现有实验步骤创建或更新接口。

### 2. 同步到题库流程

若右栏勾选了“保存时同步创建到题库”，则在实验步骤保存成功后继续执行：

1. 使用同一份共享编辑态生成题库 draft。
2. 由题库 adapter 转为 `SaveQuestionPayload`。
3. 自动生成题库编码。
4. 调用题库创建接口创建题目。

### 3. 同步后的语义

本次明确采用：**首次同步后独立维护**。

即：

- 本次同步仅创建一个新的题库题目。
- 之后再次编辑该实验步骤，不自动覆盖题库题目。
- 页面不承诺建立持续绑定。

因此 UI 文案必须避免使用“绑定题库”“联动更新”等措辞。

### 4. 错误处理

必须明确区分两类成功/失败：

- 若实验步骤保存失败：整个操作失败。
- 若实验步骤保存成功，但题库同步失败：必须提示“实验步骤已保存，但同步到题库失败”，不能回滚前者的成功信息。

## 七、与现有契约的关系

### 1. 实验步骤接口

当前前端使用：

- `createTeacherLabStep(labId, payload)`
- `updateTeacherLabStep(stepId, payload)`
- `deleteTeacherLabStep(stepId)`

本次设计不依赖改动这些接口的字段结构。

### 2. 题库接口

题库同步使用现有：

- `createQuestion(payload)`

本次同步创建不要求后端新增“绑定实验步骤”能力。

### 3. 为什么不做持续绑定

虽然 `LabStepItem` 中已有 `questionId?: number | null` 字段，但当前实验步骤创建/更新 payload 中并没有写回该字段的能力。为了避免设计与现有契约冲突，本次不把“持续绑定”作为第一阶段目标。

## 九、验证策略

### 1. 布局验证

- 页面外层不再整页滚动
- 左中右三栏可同时显示
- 三栏各自独立滚动

### 2. 编辑行为验证

- 步骤切换稳定
- 新建 / 编辑 / 删除步骤后选中状态合理
- 不同题型切换时，中栏编辑器正确刷新

### 3. 复用验证

- 题库页与实验步骤页对主要题型的交互行为保持一致
- 但两边保存 payload 仍各自正确

### 4. 同步题库验证

- 不勾选：只保存实验步骤
- 勾选：先保存实验步骤，再创建题库题目
- 第二段失败时，提示必须精确反映部分成功状态

---

## 十、范围边界

本次明确**不包含**以下内容：

1. 学生端实验作答页重构
2. 学生端代码编辑器增强
3. 实验步骤与题库题目的持续绑定或联动覆盖
4. 后端新增 questionId 绑定写回能力
5. 代码题自动判分、运行、编译、测试用例管理

---

## 结论

本次重构应当被视为一次“教师端实验步骤配置工作台升级 + typed-editor 共享层建设”。

最终结果应满足：

- 实验步骤页成为真正的三栏高密度工作台
- 页面不再依赖整页滚动
- 题型编辑交互与题库页高度统一
- 保存实验步骤时可选同步创建到题库
- 同步到题库仅影响本次保存，不引入持续绑定复杂度

这是在当前主代码契约下最接近“近乎完全复用”的第一阶段落地方案。
