# 02 系统架构与设计模式

## 1. 总体架构

系统采用前后端分离架构：

- 前端负责页面路由、角色壳层、状态管理和交互呈现
- 后端负责认证授权、业务编排、数据校验与聚合接口
- 数据库通过 Flyway migration 管理结构演进
- `design/` 提供原型解释层与页面契约层

## 2. 前端层

根据仓库结构与项目说明，前端使用：

- Vue 3
- TypeScript
- Vite
- Pinia
- Vue Router
- Element Plus

前端主要负责教师端 / 学生端双壳层页面组织、路由跳转、登录态维护和统一 API 调用。

## 3. 后端层

后端基于 Spring Boot 3 构建，主要组合为：

- Spring Web
- Spring Security
- JWT 认证
- Spring Data JPA
- Flyway

后端按业务域分包，典型结构为：

- `auth`
- `clazz`
- `student`
- `material`
- `question`
- `lab`
- `homework`
- `exam`
- `analysis`
- `plagiarism`

## 4. 数据层

数据库不依赖运行时自动建表，而是通过 migration 驱动演进。当前数据层一方面承载事务型业务数据，例如实验提交、考试提交、作业提交，另一方面也承载分析型数据，例如 `score_record` 和 `plagiarism_task`。

## 5. 可在答辩中明确说明的设计模式

### 5.1 分层架构

控制器、服务层、仓储层、实体层职责分离，控制器多以“获取当前用户 + 转发到 service”为主要模式。

### 5.2 RBAC / 角色隔离

通过 JWT + `@PreAuthorize` 实现教师与学生角色边界控制。

### 5.3 统一响应与异常处理

后端使用统一响应封装与集中异常处理，便于前端一致解析。

### 5.4 数据库版本化迁移

使用 Flyway 管理数据库变更，保证数据库结构可追踪、可复现。

## 6. 推荐答辩表述

这套架构不是单纯为了“技术先进”，而是为了支撑教学平台的三类核心复杂性：多角色权限控制、实验/作业/考试三条业务闭环，以及基于真实提交记录的统计分析反馈。
