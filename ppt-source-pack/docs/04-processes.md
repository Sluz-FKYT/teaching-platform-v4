# 04 业务流程与泳道图说明

## 1. 流程图设计原则

本素材包中的泳道图主要服务于答辩表达，因此重点突出：

- 教师与学生的职责边界
- 前端与后端之间的请求/反馈关系
- 关键状态流转
- 数据如何沉淀到提交记录、成绩记录和分析结果中

## 2. 实验闭环

主线为：

1. 教师创建实验
2. 教师配置实验步骤
3. 学生查看实验详情并逐步作答
4. 学生保存草稿答案
5. 学生正式提交实验
6. 系统执行自动评分或建议评分
7. 教师查看报告并打分
8. 系统写入成绩记录

## 3. 作业闭环

主线为：

1. 教师创建作业并发布
2. 学生查看作业详情
3. 学生提交作业内容
4. 系统保存提交记录并关联查重任务
5. 教师查看提交列表与提交详情
6. 教师批改并给出反馈
7. 系统沉淀成绩记录

## 4. 考试闭环

主线为：

1. 教师创建考试并配置试题
2. 教师发布考试
3. 学生开始考试
4. 系统创建考试提交并计算截止时间
5. 学生交卷
6. 系统自动评分客观题
7. 教师批改主观题并确认总分
8. 学生查看考试结果

## 5. 统计分析链路

统计分析并不是独立孤立模块，而是建立在实验、作业、考试提交与评分记录之上：

1. 各业务模块完成提交与批改
2. 系统写入 `score_record`
3. 系统结合 `plagiarism_task`、`score_record`、实验/作业/考试主表进行聚合
4. 教师工作台与分析页展示摘要卡、趋势、分组统计
5. 学生工作台与成绩页展示个人成绩摘要和反馈信息
